Blueprint of a Class

(November 8, 2015)


The class is an important concept in object oriented programming. It is a blueprint in which objects are created which give the object instructions for functions and attributes. For example, take a car it has wheels, a body frame, steering wheel etc… all of these are different shapes and sizes are attributes. Your car can drive, break, honk, and turn on lights all of these are examples of functions. In Ruby we create classes so that a collection of methods and variables can be encapsulated into one so that any object that invokes the class can use and access its methods and variables.


In figure 1 we have a class called Person. It has four variables, called instance variables, first, middle, last and birthday. It also has two methods initialize that takes 3 arguments and print_info which returns a string. Let’s start with the initialize function which is also called a constructor. The method must receive four strings and it assigns it to each instance variables. Instance variables are variables that start with the “@” sign and are used inside the class. The last method we have is called print_info and its purpose is to display the information in the variables.


With the four variables we list them in attr_accessor, attr_reader, or attr_writer. We are using attr_accessor for first, middle, and last variables because we want to be able to read and write to those variables. For birthday we only want to be able to read this because you cannot change the day you were born.


Lets go outside the class so we can see how to use it. The first line creates an object called person by invoking the Person class. To create the new object we use the .new operator followed by our four required arguments to pass. Here we pass in “John Dover Smith April 5, 1988” to our class variable. This says that the instance object of person holds the value “John” in variable first, “Dover” in variable middle, and so on.



To access any methods in the class we can use the object_name.method_name; for example, in the next line we run the print_info method which outputs the two string lines (see figure-2). We then access the first variable and display it to the console. The following line we can reassign the first name variable to a new name. If we try the birthday it will not work because it is an attr_reader and can only be read. I hope you were able to follow a long and see the basic pieces of classes.
