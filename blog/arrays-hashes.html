Storing Data in Ruby

(October 24, 2015)

When coding in ruby there will be times where you want to store a collection of data for later use. Two objects that can help you out with this are arrays and hashes. Both provide a way to store and access data but each has different rules on how you can use them. In my opinion, from what I can foresee is that arrays are good when it comes to storing vast amounts of computational data; for example, statistics, grades, and any other data that has to do with numbers. Hashes on the other hand, have a great use case on a particular subject and the key-value pairs are unique to that subject; for example, take and employee which has a name, address, age, department, title etc... It’s probably best to show you.

Let’s start with the array; see figure 1. You can create an array to store numbers, strings, objects or all of them in one single array. To define an array you put brackets around the data. Make sure you give the array a name that is descriptive of what data it is going to house.

Arrays store the data in a set order and assign an index number starting with 0 to each element (value); see figure 2. In order to access the stored data you need to use the index number associated with the data point you want; see figure 3. To access the value you simply type the name of the array and in the brackets you specify the index number. You can also reassign a new value this way if you needed to.

Hashes have a different setup structure as you can see in figure 4. In order to define a hash you use curly braces and then define a key value followed by an assignment and greater than operator (also known as the rocket) and last you specify a value. After each key/value pair, except for the last pair, you need to add a comma. Your keys must be unique per hash and you cannot have two of the same keys in a single hash otherwise the interpreter will return an error because it does not know which key value pair you are trying to access.

To access the data in a hash you need simply need to specify the hash name and then the key name in between the brackets.

This is only the basics when it comes to storing a collection of data in ruby. See below for some more interesting references on array and hashes.
