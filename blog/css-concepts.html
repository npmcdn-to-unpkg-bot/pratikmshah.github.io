The Definitive Guide Id & Class

(October 16, 2015)

In this modern day people expect your webpage to be visually appealing unlike the dull site of Berkshire Hathaways. In order to achieve this people use a markup language called Cascading Style Sheets; CSS for short. This language works in conjunction with HTML to add some color and delight to your site.

CSS consist of four basic parts: the selector, declaration, property and value. In figure-1 the selector is h2, the brackets are the declaration, font-size and margin-bottom are the property, and the 18pt/15px are the values. The selector is the key value that tells the html file which tags on the webpage need to be invoked with the specified CSS properties.

Although, there are several different types of selectors the two most widely used are Id (short of identifier) and Class. Each have their own purpose and syntax but also share some similarities. In figure-2 the left CSS statement shows an example of an Id which starts with a hash tag. To the right you see an example of a class which starts with a period. Those are the operators to use in order to use an identifier or class. When naming the selector at all cost avoid using numbers, abbreviate, and the name should reflect the purpose of the statement; for example, .bg-advert-bx could be used for a big advertisement box.

The Id selector is a unique identifier that can only be used once per html page. Meaning you can only use it for one element of the html page and cannot reuse it again in other parts. The use case for an id would be if you have a part of the webpage that is unique, such as a navigation bar or want to target specific areas of the page: the top, middle, or bottom then it would be valid to use an id. Another use for Ids is that it can override a class. For example, if I had a class that changes the font color to blue and I create an id that changes the color to red and use both the id supersedes the class and the font-color will be red. However, after searching on this I am coming to a universal convention that ids should rarely if not never be used.

The class selector is just like the identifier except that it allows for reusability. You can reuse the classes in as many html elements you want. A great use case for this is that if you have several different sections but want a similar look to all of them then the class is the perfect selector to use. If an id and class have conflicting properties then the class will be overridden. One consensus in the developer community is that classes are the preferred selector in CSS and all others should not be used.
