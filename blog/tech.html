My Way Your Way of Sorting

(November 19, 2015)

Important for any set of data is the ability to sort it. Although these algorithms run behind the scenes, it is an integral part for any program that stores and processess information. Sorting is what made Google such a dominant search engine; besides AdWords of course. They introduced a sorting algorithm for search engines called PageRank which ranked websites in order of most importance by the number of links that linked to the site. Today I am going to introduce you to 3 simple sorting algorithms that are used by programmers; bubble sort, insertion sort, and merge step. I also accompanied an online video after each sort algorithm to help.


Bubble Sort aka Big O

The bubble sort is the simplest of the sorting algorithms. The bubble sort works like this, you compare each pair of adjacent elements and swap if they are out of order. You continue to loop through the array until the whole array is sorted from low to high. See below the visual example (el is short for element). Video 


             
[1, 5, 3, 7, 4] // compare element 1 and 5. 
                // if el 1 is < el 0 swap else next el.
[1, 3, 5, 7, 4] // compare el 1 to el 2 el 1 < el 2. Swap
[1, 3, 5, 4, 7] // continue & repeat until array low-high
       …
[1, 3, 4, 5, 7]

Insertion Sort

The insertion sort is similar to the bubble sort. In the insertion sort an element is compared to the next adjacent element to see if the value is lower than the current value. If it is lower the elements are swapped and the new swapped element is then compared to each lower element (so el 5 compared to 4, 3…) until the element is the greater than the element to the left of it and lower than the element to the right (ex: 1 < 2 < 3). This algorithm is faster than the bubble sort because it has to make fewer comparisons. Video 


             
[3, 5, 1, 9, 6] // compare 3 and 5 - already sorted. 
[3, 1, 5, 9, 6] // now compare 5 & 1 and swap them. 
[1, 3, 5, 9, 6] // stick with el 1 and compare 3 & 1.
    …          // Move 1 to the 1st el.
[1, 3, 5, 6, 9] // repeat until array sorted low to high.

Merge Step

This last algorithm called merge sort can be a little complex but it breaks the array into smaller arrays and sorts each subarray. Once each subarray has been fully sorted they merge and repeat the sorting. This continues until the last merge where the array will be sorted.The reason for this is that sorting mini sub arrays is much faster than one big array. Video 


             
// take the 8 el array & break them into 1 el arrays
[10, 9, 15, 20, 17, 12, 5, 2] 

// then we merge and sort the mini subarrays
[10] [9] [15] [20] [17] [12] [5] [2]   

// we continue and merge once more and sort
[9, 10] [15, 20] [12, 17] [2, 5]  
[9, 10, 15, 20]  [2, 5, 12, 17]
               …
[2, 5, 9, 10, 15, 17, 20]
